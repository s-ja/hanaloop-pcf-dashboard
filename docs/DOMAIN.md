# DOMAIN.md — 탄소 도메인 개념 정리

> 본 문서는 (주)하나루프 프론트엔드 개발자 채용 과제 수행을 위해 탄소 회계 및 PCF(제품탄소발자국) 도메인을 정리한 자료입니다. 코드의 변수명·타입 설계·UI 용어 선택의 근거가 되며, 발표 시 "도메인 이해(평가 비중 25%)" 항목의 서면 자료로 활용됩니다.

## 2. 핵심 개념

### 2-1. PCF (Product Carbon Footprint, 제품탄소발자국)

**정의:** 제품 1단위를 생산·유통·사용·폐기하는 전 과정(Life Cycle)에서 발생하는 온실가스 배출량의 합계.

- **단위:** `kgCO₂eq` (이산화탄소 환산 킬로그램) — CH₄, N₂O 등 여러 온실가스를 CO₂ 기준으로 환산
- **국제 기준:** ISO 14067, GHG Protocol Product Standard
- **표기 변형:** `kgCO₂eq`, `kgCO₂e`, `kg CO2-eq` — 모두 동일한 의미이며 산업 표준 문서·국가별 가이드라인에서 혼용됨. 계산 결과와 행정 보고에 차이가 발생하지 않으므로 **표기만 통일하면 충분**함. 본 과제에서는 엑셀 원본에 쓰인 표기인 **`kgCO₂e`** 를 코드 및 UI 전체에서 일관 사용

**계산 공식**

```
PCF = Σ (활동량 × 해당 배출계수)
```

**과제 데이터 검증 예시 — 2025년 5월 (CT-045 컴퓨터 화면)**

```
전기:   (120 + 101) kWh × 0.456 kgCO₂e/kWh = 100.776 kgCO₂e
원소재: (424 + 232) kg  × 2.3 + 40 kg × 3.2  = 1,636.8 kgCO₂e
                              ↑                ↑
                         플라스틱 1         플라스틱 2
운송:   (123 + 12) ton-km × 3.5             = 472.5  kgCO₂e
─────────────────────────────────────────────────────────
5월 합계                                    = 2,210.076 kgCO₂e
```

> 위 예시는 `pcf-calculator.ts` 의 회귀 테스트(regression test) 기준값으로 사용 가능합니다.

### 2-2. GHG Scope 1 / 2 / 3

GHG Protocol이 정의한 배출원 분류 체계. **"어디서 배출이 발생하느냐"** 가 분류 기준입니다.

> **본 과제의 매핑 결정** 과제 엑셀 데이터에는 활동 유형(전기/원소재/운송)만 명시되어 있고 Scope 분류는 별도로 표기되어 있지 않습니다. 본 구현에서는 **일반적인 GHG Protocol 분류 기준 에 따라 아래와 같이 매핑**합니다. 이는 과제 필수 요구사항은 아니지만 실제 플랫폼 운영을 고려한 완성도 측면의 결정이며, UI에 `ScopeTag` 컴포넌트로 노출 하여 도메인 이해를 시각적으로 표현합니다.

|Scope|정의|본 과제 데이터 매핑|예시|
|---|---|---|---|
|**Scope 1**|기업이 직접 통제하는 배출원|(과제 데이터 미포함)|자사 보일러 연소, 사내 차량 연료|
|**Scope 2**|기업이 구매한 에너지(전기·열·증기)에서 간접 발생|**전기 사용 데이터**|공장 전력 사용 → 발전소 배출|
|**Scope 3**|가치사슬 전체의 기타 간접 배출|**원소재 + 운송 데이터**|원자재 생산(업스트림), 협력사 배송|

**프론트엔드 변수명 설계 시사점**

```typescript
type EmissionScope = 'scope1' | 'scope2' | 'scope3_upstream' | 'scope3_downstream'
```

UI에서는 `ScopeTag` 컴포넌트로 Scope 라벨을 시각화하여, 비전문가도 데이터의 출처를 직관적으로 파악할 수 있게 합니다.

### 2-3. 배출계수 (Emission Factor)

**역할:** 활동 데이터(전력량, 연료량, 중량 등)를 CO₂ 배출량으로 변환하는 계수. 플랫폼 계산 정확도의 핵심 참조 데이터입니다.

**버전 관리가 필요한 이유**

1. **정부 고시 갱신** — 한국 전력 배출계수는 환경부 고시로 주기적으로 변경됨
2. **규제별 기준 상이** — CBAM용과 국내 배출권거래제용 계수가 다를 수 있음
3. **소급 재계산 필요** — 과거 데이터를 새 계수로 재산정해야 보고서 일관성 유지

> 과제 엑셀 원문에 명시된 가이드: _"💡 배출계수는 DB에 별도 테이블로 관리하고 버전 이력을 추적하도록 설계하세요."_

본 과제 구현에서는 `EmissionFactor` 인터페이스에 `version`, `validFrom` 필드를 포함하여 버전 관리 개념을 코드에 반영합니다. 구체적 필드 정의는 `PLANNING.md` 의 데이터 모델 섹션을 참조하세요.

> **본 과제에서 다루지 않는 영역**
> 
> - 배출계수 출처별 분류 (IPCC, MOE, ECOINVENT 등): 과제 데이터에 출처 정보가 명시되어 있지 않으므로 본 구현 범위에서 제외합니다.
> - 공급업체별 실측 배출계수: 과제 데이터에 해당 정보가 없으며 단일 기본값 계수만 제공됩니다.
> - 위 두 항목은 실제 플랫폼에서는 필요한 기능이나, 본 과제에서는 데이터가 제공되지 않아 인터페이스 확장 여지만 남겨두는 수준으로 처리합니다.

### 2-4. 과제 데이터에서의 PCF 계산 흐름

```
[원소재 입력]              [전기 입력]                [운송 입력]
투입량(kg)                전력 사용량(kWh)            중량·거리(ton-km)
    │                         │                          │
    ▼                         ▼                          ▼
× 소재별 배출계수         × 전력 배출계수            × 운송수단별 배출계수
  플라스틱1: 2.3              0.456                      트럭: 3.5
  플라스틱2: 3.2          (kgCO₂e/kWh)              (kgCO₂e/ton-km)
  (kgCO₂e/kg)
    │                         │                          │
    ▼                         ▼                          ▼
materialEmissions        energyEmissions          transportEmissions
[Scope 3 업스트림]         [Scope 2]                 [Scope 3]
    │                         │                          │
    └─────────────────────────┼──────────────────────────┘
                              ▼
                  totalPCF = 세 카테고리 합산
                              ▼
                    단위: kgCO₂e / 제품 1단위
```

## 3. 과제 데이터 기준 배출계수 (확정값)

> **본 값은 과제용 데이터 시트의 "배출계수 (지원자 참고용)" 표에서 직접 추출된 값입니다. 코드 전체에서 이 값만 사용하며, 다른 값으로 임의 대체하지 않습니다.**

|항목|계수값|단위|코드 식별자 (제안)|
|---|---|---|---|
|전기 (한국전력 기본값)|**0.456**|kgCO₂e / kWh|`EF_ELECTRICITY_KEPCO`|
|원소재 (플라스틱 1)|**2.3**|kgCO₂e / kg|`EF_MATERIAL_PLASTIC_1`|
|원소재 (플라스틱 2)|**3.2**|kgCO₂e / kg|`EF_MATERIAL_PLASTIC_2`|
|운송 (트럭)|**3.5**|kgCO₂e / ton-km|`EF_TRANSPORT_TRUCK`|

**Single Source of Truth 원칙**

- 위 4개 값은 `lib/constants.ts` 의 `EMISSION_FACTORS` 상수 객체에 단일 선언
- `lib/pcf-calculator.ts` 와 `lib/mock-data.ts` 양쪽에서 동일 import
- 값 변경이 필요할 경우 `constants.ts` 한 곳만 수정 → 계산 결과 일관성 보장

## 4. 과제 데이터 구조 (실제 데이터)

**제품:** 컴퓨터 화면 **CT-045** (2025년 1~8월 데이터)

**컬럼 구조 (엑셀 원본)**

|컬럼|타입|예시|비고|
|---|---|---|---|
|일자(원본)|string (YYYY-MM-DD)|`2025-01-01`|월 단위 집계 기준|
|활동 유형|string|`전기` / `원소재` / `운송`|카테고리 식별|
|설명|string|`한국전력` / `플라스틱 1` / `트럭`|배출계수 매칭 키|
|량|number|`110`|활동 수치|
|단위|string|`kWh` / `kg` / `ton-km`|배출계수 단위와 일치해야 함|

**중요 주의사항**

- **2025년 5월에 동일 활동유형 중복 행이 의도적으로 존재** (전기 101 kWh, 원소재 플라스틱 1 232 kg, 운송 트럭 12 ton-km)
- 이는 데이터 오류가 아니라 **"동일 월 내 복수 활동 기록"** 시나리오를 검증하기 위한 의도된 데이터로 판단됨
- 따라서 동일 `(period, category, description)` 조합에 대해 **유니크 제약을 걸지 않고**, 각 행을 별개 `ActivityData` 레코드로 저장하여 집계 시 합산하는 설계가 올바름

## 5. 용어 사전 (Glossary)

|영문|한글|본 과제에서의 의미|
|---|---|---|
|PCF (Product Carbon Footprint)|제품탄소발자국|제품 1단위당 전과정 온실가스 배출량|
|GHG (Greenhouse Gas)|온실가스|CO₂, CH₄, N₂O 등 기후변화 유발 가스|
|Scope 1 / 2 / 3|스코프 1 / 2 / 3|배출원 직간접성에 따른 분류 체계|
|Emission Factor|배출계수|활동량을 배출량으로 변환하는 계수|
|Activity Data|활동 데이터|사용자가 입력하는 원본 수치(전력 사용량 등)|
|LCA (Life Cycle Assessment)|전과정 평가|원자재~폐기까지 환경 영향 평가 방법론|
|CBAM|EU 탄소국경조정제도|EU 수출 시 탄소 배출량 보고 의무 제도|
|ISO 14067|—|제품 탄소발자국 산정 국제 표준|
|`kgCO₂e`|—|이산화탄소 환산 킬로그램 (단위 표기)|
|`ton-km`|—|운송 단위: (운송 중량 톤) × (거리 km)|
