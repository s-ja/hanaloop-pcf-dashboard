# USER_RESEARCH.md — 사용자 분석 및 시나리오

> 본 문서는 (주)하나루프 프론트엔드 개발자 채용 과제의 사용자 정의와 시나리오를 정리한 자료입니다. 과제 요구사항인 "대상자는 실무자, 경영자"를 페르소나로 구체화하고, 각 사용자의 핵심 질문과 사용 흐름을 기반으로 대시보드에 반드시 포함되어야 할 뷰/기능 목록을 도출합니다.
>
> 발표 시 "사용자 경험(UX) — 평가 비중 25%" 항목의 서면 자료로 활용됩니다. 도메인 용어와 개념은 `DOMAIN.md` 를 전제로 합니다.

---

## 1. 분석 접근 방법

### 1-1. 페르소나 정의 근거

과제 산출물 명세는 **"대상자는 실무자, 경영자"** 로 두 사용자 유형을 명시했습니다. 이 두 유형은 **동일 데이터를 보더라도 묻는 질문과 의사결정 단위가 다릅니다**.

| 구분           | 실무자                          | 경영자                         |
| -------------- | ------------------------------- | ------------------------------ |
| 시간 단위      | 일·주·월                        | 분기·연                        |
| 관심 입자      | 개별 데이터 정확성              | 총량 추세와 변동 원인          |
| 정보 처리 방식 | 입력·검증·수정                  | 시각화·요약·보고               |
| 의사결정       | "수치를 어떻게 정확히 산출할까" | "지금 우리 회사는 어디 있는가" |

이 차이가 **두 페르소나를 위한 뷰 분리(`/dashboard`, `/input`)** 결정의 근거가 됩니다.

### 1-2. 도메인 맥락 반영

`DOMAIN.md` 1-3에서 정리한 하나루프의 주요 고객사가 **제조·유통 중심**(이마트, 벽산, 두산에너빌리티, 삼양스틸 등)이라는 점, 그리고 본 과제 데이터가 **컴퓨터 화면(CT-045) 이라는 단일 제품의 월별 활동 데이터**라는 점을 고려하여, 페르소나를 "B2B 제조업 기업의 환경/경영 담당자"로 설정합니다.

## 3. 주요 사용 시나리오

### 3-1. 시나리오 1 — 월말 데이터 입력 및 즉시 검증 (실무자)

**상황:** 김환경 주임이 2025년 9월 마감일에 9월 활동 데이터를 입력해야 함.

```
1. /input 페이지 진입
2. 활동 유형 선택: "전기"
3. 일자 입력: 2025-09-01
4. 설명 선택: "한국전력" (배출계수 자동 매칭 — 0.456 kgCO₂e/kWh)
   → 화면에 사용될 배출계수 버전 배지 표시 ("KEPCO 2025")
5. 활동량 입력: 125, 단위는 kWh로 자동 표시
6. 제출 → 즉시 9월 PCF 카테고리별 기여도 갱신
7. 전월(8월) 대비 증감 자동 계산 → 증가 시 하이라이트
8. 동일 월 추가 데이터(원소재·운송)도 같은 흐름으로 입력
```

**필요한 UI 동작:**

- 활동 유형 선택 시 → 단위 자동 표시 + 가능한 "설명" 옵션 필터링
- 잘못된 입력(음수, 공란, 단위 불일치) 즉시 에러 메시지 ← **체크리스트 필수 항목**
- 입력 즉시 PCF 프리뷰 — 페이지 이동 없이 계산 결과 확인

### 3-2. 시나리오 2 — 임원회의 5분 전 대시보드 조회 (경영자)

**상황:** 이탄소 이사가 회의 직전 자사 PCF 현황을 빠르게 확인.

```
1. /dashboard 페이지 진입
2. 최상단: "2025년 누적 PCF: 12,345 kgCO₂e (전년 동기 대비 -8%)" 큰 글씨로 즉시 인지
3. 카테고리별 파이차트: 원소재가 70% 차지 → 감축 우선순위 즉시 파악
4. 월별 트렌드 라인차트: 5월에 PCF 급등 → 클릭하면 5월 상세로 이동(드릴다운)
5. 목표 진척도 게이지: 연간 목표 대비 65% 진행 → 빨간색 경고 표시
6. 임원회의용 캡처: 화면 우상단 "이미지로 저장" 또는 깔끔한 인쇄 레이아웃
```

**필요한 UI 동작:**

- 첫 화면에서 핵심 정보 3가지(총량/구성/목표)를 한 화면에 노출
- 차트는 마우스 오버 시 정확한 수치 표시 — 발표·보고 시 신뢰성 핵심
- 인쇄/캡처 친화적 레이아웃 (배경 단순, 차트 명도 대비 명확)

### 3-3. 시나리오 3 — 데이터 이상 발견 후 원인 추적 (실무자)

**상황:** 대시보드에서 5월 PCF가 비정상적으로 높은 것을 발견.

```
1. /dashboard 트렌드 차트에서 5월 데이터 포인트 클릭
2. 5월 상세 페이지/모달: 카테고리별 기여도 + 개별 활동 데이터 목록
3. "전기" 카테고리 클릭 → 5월 전기 활동 2건 확인 (120 kWh + 101 kWh)
   → 동일 월 중복 입력이 정상인지 확인 (정상 케이스로 판정)
4. "원소재" 카테고리 클릭 → 5월 원소재 활동 3건 확인
   → 플라스틱 1: 424 kg, 232 kg (중복) + 플라스틱 2: 40 kg
   → 232 kg 입력이 추가 발주분으로 정상 확인
5. 결과: 5월 급등은 데이터 오류가 아닌 실제 생산량 증가로 결론
6. (선택) 메모 또는 라벨 추가 — 향후 동일 패턴 확인 시 참고
```

**필요한 UI 동작:**

- 트렌드 차트의 각 데이터 포인트는 클릭 가능(인터랙티브 필수 요구사항)
- 드릴다운 시 카테고리 → 개별 활동 데이터 순으로 깊이 확장
- 동일 월 복수 활동 데이터를 **합산값과 개별값 모두** 확인 가능해야 함 ← `DOMAIN.md` 4장의 중복 데이터 처리 결정과 직결

## 4. 대시보드 필수 포함 뷰/기능 목록

### 4-1. 우선순위 정리

우선순위는 **프로덕트로서의 완성도** 관점에서 평가합니다. 페르소나가 실제로 해당 기능 없이 자기 목적을 달성할 수 있는지를 기준으로 합니다.

| 우선순위 | 뷰/기능                                                | 주 대상 페르소나 | 위치                            |
| -------- | ------------------------------------------------------ | ---------------- | ------------------------------- |
| ⭐⭐⭐   | **총 PCF 수치 카드** (현재값 + 전기간 대비 증감)       | 경영자 + 실무자  | `/dashboard` 최상단             |
| ⭐⭐⭐   | **카테고리별 배출 기여도** (전기/원소재/운송 파이·바)  | 경영자 + 실무자  | `/dashboard`                    |
| ⭐⭐⭐   | **데이터 입력 폼** (활동 유형/일자/량/단위 + 검증)     | 실무자           | `/input`                        |
| ⭐⭐⭐   | **시계열 트렌드 차트** (월별 PCF 추이, 인터랙티브)     | 경영자 + 실무자  | `/dashboard`                    |
| ⭐⭐⭐   | **에러 메시지 표시** (입력 오류 시)                    | 실무자           | `/input`                        |
| ⭐⭐⭐   | **단위 표시 일관성** (`kgCO₂e`, `kWh`, `kg`, `ton-km`) | 양쪽             | 전 영역                         |
| ⭐⭐     | **목표 대비 진척도** (Progress / Gauge)                | 경영자           | `/dashboard`                    |
| ⭐⭐     | **배출계수 버전 배지** (사용된 계수의 출처·시점 명시)  | 실무자 + 경영자  | 차트·테이블 인근                |
| ⭐⭐     | **기간 필터** (월별/분기별/연간)                       | 양쪽             | `/dashboard` 상단               |
| ⭐⭐     | **Scope 라벨** (Scope 2 / Scope 3 배지)                | 양쪽             | 카테고리 컴포넌트 내            |
| ⭐       | **드릴다운 — 카테고리에서 개별 활동으로**              | 실무자           | `/dashboard`                    |
| ⭐       | **활동 데이터 목록 테이블**                            | 실무자           | `/dashboard` 하단 또는 `/input` |

**우선순위 기준 정리**

- **⭐⭐⭐ (필수):** 이 기능이 없으면 페르소나가 본인의 목적을 달성할 수 없거나, 체크리스트 필수 항목을 충족하지 못함. 1차 구현 범위.
- **⭐⭐ (권장):** 기능이 없어도 핵심 흐름은 동작하나, 프로덕트로서의 완성도와 사용성에 큰 영향. 시간 허용 시 1차에 포함.
- **⭐ (선택):** 사용 흐름의 일부 케이스(이상값 추적 등)에서만 필요. 보너스 영역.

### 4-2. 평가 체크리스트와의 매핑

지원자 체크리스트의 필수 항목과 위 기능의 대응 관계:

| 체크리스트 항목                                           | 대응 기능                                              |
| --------------------------------------------------------- | ------------------------------------------------------ |
| PCF 계산 결과가 시각화, 직관적으로 이해 가능              | 총 PCF 수치 카드 + 카테고리별 차트 + 트렌드 차트       |
| 데이터 값이 정확하고 단위 표시 적절                       | 모든 수치에 단위 명시(`kgCO₂e`, `kWh`, `kg`, `ton-km`) |
| 데이터 입력 화면에서 오류 입력 시 에러 메시지 표시        | 데이터 입력 폼의 클라이언트 검증 + 에러 메시지         |
| 비전문가도 직관적으로 데이터 입력하고 결과를 읽을 수 있음 | Scope 라벨 툴팁, 단위 자동 표시, 입력 즉시 프리뷰      |

## 5. UX 핵심 결정 요약

| 결정                                                                   | 근거                                                                   |
| ---------------------------------------------------------------------- | ---------------------------------------------------------------------- |
| **두 페르소나 뷰 라우트 분리** (`/dashboard`, `/input`)                | 정보 우선순위 차이가 명확하며, 경영자에게 입력 폼은 노이즈             |
| **대시보드 최상단에 총 PCF + 증감률 노출**                             | 경영자가 짧은 세션 시간 내 핵심 정보를 먼저 인지할 수 있어야 함        |
| **모든 차트는 인터랙티브**                                             | 과제 산출물 요구사항인 "인터랙티브 대시보드" 직접 대응                 |
| **단위·Scope 라벨을 시각적 컴포넌트로 분리** (`UnitLabel`, `ScopeTag`) | "비전문가도 이해할 수 있는 UX"(체크리스트 필수) 직접 대응              |
| **배출계수 버전 배지 노출**                                            | 실무자가 사용된 계수의 출처·시점을 추적할 수 있어야 데이터 신뢰성 확보 |
| **동일 월 중복 데이터를 합산값+개별값 양쪽으로 확인 가능**             | 과제 데이터의 5월 중복 행 처리 결정(`DOMAIN.md` 4장)과 일관            |
| **드릴다운 동선 제공**                                                 | 실무자의 이상 데이터 추적 시나리오(시나리오 3) 직접 지원               |

## 6. 비전문가 대응 UX 체크포인트

체크리스트 필수 항목인 _"비전문가도 직관적으로 데이터를 입력하고 결과를 읽을 수 있는가"_ 에 대응하기 위한 세부 결정:

- **용어 보조** — Scope, PCF, 배출계수 등 도메인 용어 옆에는 `?` 아이콘 툴팁으로 한 줄 설명 제공
- **단위 자동 표시** — 활동 유형 선택 시 단위 입력란이 자동으로 적절한 단위를 표시, 사용자가 단위를 잘못 선택할 여지 차단
- **색상으로 신호** — 증가는 빨강, 감소는 초록 (탄소 배출 맥락에서 직관적). 단, 색맹 접근성을 위해 아이콘(↑/↓)도 함께 표시
- **단위·통화 명확성** — 모든 수치는 단위와 함께 표시. 천 단위 구분자(`12,345`) 적용
- **로딩·에러 상태** — 빈 데이터, 로딩 중, 에러 상황 각각의 UI 명시적 처리

## 7. 본 과제 범위에서 다루지 않는 사항

| 항목                              | 사유                                               |
| --------------------------------- | -------------------------------------------------- |
| 다중 사용자 권한 / 인증           | 과제 데이터는 단일 제품(CT-045)이며 인증 요구 없음 |
| 협업 기능 (코멘트, 공유)          | 과제 범위 밖, 발표 시 확장 방향으로 언급 가능      |
| 다국어 지원                       | 과제는 한국어 기반, 인터페이스 확장 여지로만 남김  |
| AI 어시스턴트(카본버디 같은 기능) | Hana.eco 실제 제품 기능이나 과제 범위 밖           |

> **타 시스템 비교 기능에 대하여** 지원자 체크리스트의 보너스 항목 중 "타 시스템과 비교"가 포함되어 있습니다. 본 과제 데이터에는 단일 제품(CT-045)만 제공되지만, 비교 기능 구현 여지는 데이터 모델(`Product` 인터페이스, 다중 제품 지원 구조)에서 열어두며, 시간 여유 시 보너스 항목으로 시도합니다.

위 항목들은 발표 시 "현재 구현 범위 vs. 확장 방향"의 trade-off 설명 소재로 활용.
